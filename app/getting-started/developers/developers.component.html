Developer Onboarding

Summary

The Rivet Design System is a collection of standardized components and visual styles (icons, colors, typography). Built in collaboration with our development and design teams, Advicent can now quickly create beautiful, simple, and consistent user experiences across all our products. As a developer, you will use this document site to speed up implementation with reusable code snippets and defined UI standards. Check out our Mission and Values to learn more about this effort and see our Changelog to stay up to date with the latest releases.

Rivet workflow

    This is an overview of the development process using Rivet:
  1. Complete local developer setup on your computer
  2. Include Rivet in your build
  3. Visit Rivet document site to copy component, variable and mixins
  4. Customize component code by referring to Rivet documentation, consulting your development team, or reaching out to the DesignOps team
  5. Refine component implementation with code and design reviews
  6. Deploy with a pull request

What is included

    To make the Rivet Documentation site valuable to your workflow, each UI component and visual style includes the following
  • Name, description, and usage
  • Interactive examples with resuable code snippets
  • Parameters for code customization
  • Design specification documentation for reference

Implementing Rivet

Within the projects package.json and rivet-style: 'x.x.x' under devDependencies, using the version of Rivet you want to import

Using Rivet

Below are examples of how to use components, icons, colors, and typography.

Components

There are a variety of components ready to be embedded within projects. Each hosted component comes with documentation explaining how to insert the component as well as different styles and configurations available. To see quick examples of every component, visit the components overview page.

Icons

Icons are built out within the Rivet package into an icon font using uploaded svgs. This font allows for a standard icon that can be scaled and styled easily across products. To view all available icons, and more information on implementation, visit icons.

Colors

Color can be set using defined color variables. We use CSS variables for ease of implementation across products and code styles. For a full list of color variables see colors.

Typography

Rivet has set variables to use for font styles in order to ensure consistency. For examples and a list of variables check out font styles.

Making customizations

If the component options available within a Rivet component are not enough to achieve a given UI, a developer should consult the DesignOps team to determine the best way to move forward. Reach out through Teams.

Contributing

We encourage developer contributions because they make the Rivet Design System relevant, stable, and flexible. See our contribute page for an overview of the contribution process.

Questions or feedback

For any questions, reach out to the DesignOps team through Teams. Feature requests can be added in Azure.
