Contributing to Rivet

Making changes to Rivet

To make changes to the Rivet source code, follow the instructions listed here.

Updating Rivet version

Once changes are made to the rivet-style package (documentation only updates do not need version updates), the version number must be incremented in
rivet-style > package.json > "version": "x.x.x";.

Versioning format

    Rivet versioning follows NPM versioning standards. (Major).(Minor).(Patch)
  • (Major).X.X - Major releases are used for large UI overhauls and systematic breaking changes
  • X.(Minor).X - Minor releases are for new component additions and other new features
  • X.X.(Patch) - Patch releases are reserved for bug fixes and minor styling adjustments

You can find more information on NPM versioning here

Updating the Changelog

When updating the version number, it is required to update the changelog located in the documentation site.
You can find the changelog under app > changelog > versions > versions.component.ts
