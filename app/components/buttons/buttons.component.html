Buttons

Buttons are clickable elements that emphasize actions users can take on the page.

Use Case

Use Buttons in dialogs, forms, settings, and cards. Buttons should always follow this hierarchy:
Primary > Secondary > Tertiary > Text.

Implementation

Primary Buttons

Use Case

Use Primary Buttons to give actions high emphasis. Display only 1-3 Primary Buttons to effectively direct attention to primary actions on a page.

Secondary Buttons

Use Case

Use Secondary Buttons to give actions medium emphasis. Use Secondary Buttons for important actions that are not considered primary actions.

Tertiary Buttons

Use Case

Use Tertiary Buttons to give actions lower emphasis. Use Tertiary buttons for actions that need to be differentiated from static text.

Text Buttons

Use Case

Use Text Buttons to give actions low emphasis. Text Buttons are less noticeable and commonly found in dialogs, cards, or next to higher emphasis buttons.

Icon Buttons

Use Case

Use Icon Buttons for one-click actions that can be widely recognized by just the icon.

Implementation

When using icons within buttons, it is required to add aria-label="icon-type" for screen readers to understand the button does.

Button with Icon

Use Case

Use Buttons with Icons to emphasize one-click actions with familiar icons.

Implementation

When using icons within buttons, it is required to add aria-label="icon-type" for screen readers to understand the button does.

Loading Buttons

Use Case

Use Loading Buttons when a button initiates an action and takes over 1 second to confirm the action with another form of feedback. For more information, see loading

Implementation

If the button does not contain text saying "loading" and only contains an icon, add aria-label="loading" for screen reader support.
