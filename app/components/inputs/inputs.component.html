Input

Inputs capture and display data entered by the user.

Use Case

Use Inputs differently depending on if they have a label. Inputs with labels are used for standalone inputs, where the labels refer to what the single input is asking for. Inputs without labels are used for multiple/form inputs as the form itself will contain the labels.

Implementation

Inputs currently use the Angular Material input component and directives as their basis.

Input states

Invalid

Invalid styles can either be applied as a conditional class of [class.invalid]="boolean" applied to the mat-form-field container OR through form control set on the input, OR through the use of thervt-form-field directive set on the mat-form-field container.

Disabled

For disabled styles, simply set a conditional disabled attribute on the input or use the disabled state in form control. Material will apply the classes to the rest of the tags automatically.

Read-only

Since Material and form control do not support read-only states, a conditional class of [class.read-only]="boolean" needs to be applied to the mat-form-field container. The input must also be disabled in order to use the read-only class.

Input sizes

    There are 4 input size classes available to use.
  • rvt-input-sm : 45px wide. Used for inputs with a max of 3-4 characters such as ages or percentages.
  • rvt-input-md : 135px wide. Can be used for full date format or any of the above.
  • rvt-input-lg : 210px wide. Used for full names or any of the above.
  • rvt-input-full : 100% width of container. Used for long inputs or for matching styles.
