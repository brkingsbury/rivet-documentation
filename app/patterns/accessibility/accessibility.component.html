Accessibility

What is it?

The practice of making your site accommodating to everyone regardless of situation, circumstances or ability.

    There are 5 common accomodations to consider when building out products accessibly.
  • Visual - Low or no Vision, Color blindness
  • Auditory - Deafness or hard of hearing
  • Physical / Motor - Limited mobility, tremors
  • Cognitive - Dyslexia, autism, English as a second language
  • Technical - Small screens, old technology, connectivity, internet speeds

Our Approach

In order to make our products easier to use for everyone, we follow accessibility recommendations from the Web Content Accessibility Guidelines 2.1 - AA [WCAG 2.1].

Principles

There are 4 principles that the WCAG guidelines follow to set a foundation for anyone to access and use web content.

Perceivable

All users are able to see or hear the content on the page.

Operable

All users are able to interact with the interface and navigation.

Understandable

All users are able to comprehend the content of the site.

Robust

All users can use the site with assistive techonologies (screen readers, screen magnifiers, etc.)

Guidelines

Color Contrast

Across our products, we meet the WCAG contrast success criterion 1.4.3. This means that all text must have a minimum contrast ratio of 4.5:1.

Color contrast can be tested at accessible-colors.com

Focus states

For focus states we use our hover state styling with the addition of the browser default outline in order to draw attention to the component for keyboard users.

ARIA Tags

For custom components, created with non standard elements, we use ARIA tags to enable screen-readers to understand what the component is.

Testing

To verify that we are following accessibility standards, there are multiple ways to test our products.

Axe Developer Tools - a browser extension for testing any site. Available for Chrome + Firefox.
Firefox has built in accessibility testing under web developer > accessibility (Shift + F12).
Chrome has built in accessibility testing under DevTools > Audits.
WAVE - a browser extension for testing accessibility errors. Availble for Chrome + Firefox.
NVDA - free software (for windows) to test screen reader support. Available here.

Keyboard Navigation

All functionality that is accomplished with a mouse, should also be achievable using only a keyboard.

  • To advance through elements of the page, users should be able to use the TAB key.
  • Users should also be able to reverse through elements using SHIFT + TAB
  • To select an element (such as a button, link, or tab), the user can hit the ENTER key.
  • To escape an element (such as dialogs, modals, and dropdowns) users can use the ESC key.
  • Within certain elements that contain extra functionality (like dropdowns and datepickers), users should be able to navigate around using the arrow keys and select options using SPACE
  • When navigating within components (such as dialogs), focus should remain within the component until the user closes/exits the component. No background elements should be focusable while the user is in the component.
